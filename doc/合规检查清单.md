# Trading Studio 合规检查清单

> 适用于产品上线、功能发布、数据源接入、AI 能力变更和商业化评审。

## 1. 产品边界

- [ ] 首页、股票页、AI 页已明确展示“信息仅供参考，不构成投资建议”
- [ ] 未提供券商登录、自动下单、自动调仓等交易终端能力
- [ ] 未出现“必涨”“稳赚”“目标价”“买入/卖出建议”等表述
- [ ] 未上线荐股排行榜、收益承诺或变相投顾功能

## 2. 数据源与版权

- [ ] 每个新闻源已确认 robots 和抓取条款
- [ ] 每个商业数据源已确认授权范围、访问频率和使用边界
- [ ] 公告原文、新闻正文、研报摘要的保存方式符合版权约束
- [ ] 未授权的付费研报不保存全文、不对外分发
- [ ] 数据源授权材料、商务邮件或合同已留档

## 3. 行情使用

- [ ] 已确认实时/延迟行情的授权类型
- [ ] 若为延迟行情，前台明确展示延迟说明
- [ ] 未超出授权范围对外开放二次分发接口
- [ ] 行情异常时前台会标注数据时间，不伪装为实时

## 4. AI 输出合规

- [ ] Prompt 明确禁止收益承诺、目标价和直接买卖建议
- [ ] AI 输出必须包含风险声明
- [ ] AI 输出必须包含引用来源或明确标记数据不足
- [ ] 已设置输入注入、越权访问和恶意提问拦截
- [ ] 已设置输出合规审查规则和关键词黑名单
- [ ] 高风险任务支持人工抽检与回滚 Prompt 版本

## 5. 用户与隐私

- [ ] 用户协议、隐私政策、免责声明已上线
- [ ] 已声明会记录 AI 会话、使用日志和行为日志
- [ ] 敏感信息采集最小化，仅收集业务必要数据
- [ ] 登录日志、设备信息、IP 信息的使用和保留有制度约束
- [ ] 用户注销后具备数据清理或匿名化方案

## 6. 安全控制

- [ ] 全站启用 HTTPS
- [ ] 用户密码使用 Argon2id 或同等级安全哈希
- [ ] 内部接口使用 HMAC、时间戳和 nonce
- [ ] MySQL、Redis、FastAPI 未直接暴露公网
- [ ] 密钥、Token、Cookie 未明文写入日志
- [ ] 审计日志不可被普通管理员删除

## 7. 内容治理

- [ ] 新闻、公告、事件支持人工下线和屏蔽
- [ ] 已设置敏感词、风险词和高争议表达库
- [ ] 政策类、监管类、处罚类重大内容可人工复核
- [ ] 转载内容能回溯原始来源并显示出处

## 8. 商业化与营销

- [ ] 营销文案未使用误导性收益承诺
- [ ] 套餐说明中已明确额度、限制与不包含项
- [ ] API 产品说明中已明确数据源授权边界
- [ ] 面向企业客户的私有化部署条款单独审查

## 9. 运维与审计

- [ ] 数据源变更、Prompt 变更、模型变更均有审计记录
- [ ] 已建立异常事件处置流程和升级机制
- [ ] 已建立数据质量告警和值班机制
- [ ] 已执行备份与恢复演练
- [ ] 已定义重大合规事故的停服或降级预案

## 10. 发布门禁

- [ ] 本次发布涉及的数据源和接口变更已完成复核
- [ ] 本次发布涉及的 AI 能力已完成合规评测
- [ ] 样例页面已人工检查免责声明和风险提示
- [ ] 法务/业务负责人已完成上线确认

## 11. 高风险红线

以下任一项存在则不得上线：

- [ ] 无授权抓取或分发受限内容
- [ ] 输出直接荐股或收益承诺
- [ ] 未经说明展示伪实时行情
- [ ] AI 可越权访问非公开用户数据
- [ ] 后台无法审计 Prompt、模型和新闻源变更
