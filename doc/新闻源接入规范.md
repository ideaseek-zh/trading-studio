# Trading Studio 新闻源接入规范

## 1. 目的

本文档定义新闻、公告、政策信息和市场快讯类数据源的接入标准，保证不同来源在采集、清洗、入库、追踪、监控和合规方面具备统一约束。

## 2. 适用范围

- 财经媒体新闻源
- 交易所与巨潮资讯公告源
- 政策与宏观公开信息源
- 上市公司官网公告/新闻源
- 采购或合作的商业资讯接口

## 3. 接入原则

### 3.1 合法合规优先

- 必须确认来源授权、转载限制、抓取协议和 robots 规则
- 未确认商业授权前，不得将受限内容用于商业环境
- 对受版权限制的内容只保存元数据、摘要和跳转链接

### 3.2 Provider 抽象统一

所有新闻源必须实现统一的 Provider 接口，不允许业务代码直接依赖某个站点的 HTML 结构或第三方 SDK。

```python
from abc import ABC, abstractmethod
from typing import Sequence

class NewsProvider(ABC):
    provider_key: str

    @abstractmethod
    async def fetch_list(self, cursor: str | None = None) -> Sequence[dict]:
        ...

    @abstractmethod
    async def fetch_detail(self, source_item_id: str) -> dict:
        ...

    @abstractmethod
    async def health_check(self) -> dict:
        ...
```

### 3.3 全链路可追溯

每条新闻必须可追溯到源站记录、抓取时间、解析版本和原始载荷文件。

## 4. 新闻源分级

| 级别 | 定义 | 示例 | 使用策略 |
| --- | --- | --- | --- |
| `L1` | 官方披露源 | 巨潮资讯、上交所、深交所、北交所 | 最高可信度，优先入库 |
| `L2` | 主流财经媒体 | 财联社、东方财富、证券时报 | 重要新闻主来源 |
| `L3` | 宏观与政策机构 | 证监会、发改委、工信部、央行 | 政策事件关键来源 |
| `L4` | 公司官网或转载站 | 上市公司官网、新闻稿站点 | 作为补充来源 |
| `L5` | 社区与情绪源 | 雪球等 | 默认不作为事实主源，仅作舆情参考 |

## 5. 标准字段模型

### 5.1 列表页字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `provider` | string | 是 | Provider 标识 |
| `source_id` | string | 是 | 源站标识 |
| `source_item_id` | string | 是 | 源站内唯一 ID |
| `title` | string | 是 | 原始标题 |
| `canonical_url` | string | 否 | 标准化链接 |
| `published_at` | datetime | 是 | 源站发布时间 |
| `author` | string | 否 | 作者 |
| `category` | string | 否 | 分类 |
| `raw_payload` | object | 是 | 原始返回数据 |

### 5.2 详情页字段

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `content_html` | string | 否 | 原始 HTML |
| `content_text` | string | 是 | 清洗后的正文 |
| `attachments` | array | 否 | 附件信息 |
| `images` | array | 否 | 图片地址 |
| `tags` | array | 否 | 源站标签 |
| `language` | string | 是 | 默认 `zh-CN` |
| `copyright_status` | string | 是 | `public`、`restricted`、`licensed` |

- `tests/`：字段契约、样本回归和异常用例

## 7. 接入流程

```text
源站发现
  -> 授权与 robots 审核
  -> 录入 source 配置
  -> 实现 Provider
  -> 列表抓取
  -> 详情抓取
  -> 字段标准化
  -> 去重与聚类
  -> 实体识别
  -> 重要性分级
  -> AI 摘要与事件抽取
  -> 搜索索引与分发
```

## 8. 去重与聚类规范

### 8.1 去重层级

1. `source_id + source_item_id`
2. 规范化后的 `canonical_url`
3. `title_hash`
4. `content_hash`
5. `simhash` 或向量相似度

### 8.2 聚类策略

- 相似新闻不直接删除，而是归入 `news_clusters`
- 主记录用于前台展示，簇内记录保留原始来源和时间
- 聚类需要保留主从关系和合并理由

## 9. 正文清洗规范

- 删除广告、脚本、导航、相关推荐和评论区
- 保留段落顺序、关键数字、时间和主体名称
- 对表格、图片说明、附件标题进行文本化处理
- 正文不能为空；为空则标记 `quality_status=failed`

## 10. 实体识别规范

### 10.1 识别对象

- 股票代码
- 股票简称和全称
- 板块、行业、概念
- 子公司、客户、供应商
- 政策部门与监管机构
- 关键产品和技术名词

### 10.2 识别策略

- 一级：规则词典与别名库
- 二级：拼音、简称、历史名称模糊匹配
- 三级：Embedding 相似召回
- 四级：LLM 复核与歧义消解

## 11. 重要性评分规范

```text
importance_score =
  source_weight * 0.15 +
  event_type_weight * 0.25 +
  entity_relevance * 0.20 +
  market_scope * 0.15 +
  novelty_score * 0.10 +
  price_reaction_score * 0.10 +
  manual_boost * 0.05
```

| 等级 | 含义 | 处理策略 |
| --- | --- | --- |
| `S` | 全市场重大事件 | 立即推送并进入首页 |
| `A` | 重大政策/重大公司事项 | 重点展示，支持提醒 |
| `B` | 明显影响公司或行业 | 常规展示 |
| `C` | 一般资讯 | 仅收录 |
| `D` | 低价值或重复内容 | 不前台推荐 |

## 12. 数据质量要求

### 12.1 必填元数据

- `provider`
- `source_url`
- `fetched_at`
- `source_timestamp`
- `checksum`
- `request_id`
- `raw_payload_path`
- `parser_version`
- `quality_status`

### 12.2 SLA 建议

| 类型 | 目标延迟 | 成功率 |
| --- | --- | --- |
| 交易所公告 | 3 到 10 分钟 | >= 99% |
| 主流快讯 | 1 到 3 分钟 | >= 98% |
| 政策新闻 | 5 分钟内 | >= 98% |
| 公司官网新闻 | 15 分钟内 | >= 95% |

## 13. 频控与熔断

- 每个源必须设置 `rate_limit`、`retry_policy`、`timeout`
- 连续失败达到阈值时进入短期熔断
- 熔断期间切换备用源或降低刷新频率
- 后台告警必须展示最近错误码、最后成功时间和样本请求

## 14. 安全与合规要求

- 禁止绕过登录态、验证码和访问控制抓取受限内容
- 禁止保存未授权的完整付费研报正文
- 涉及 Cookie、Token 的源必须通过密钥管理系统注入
- 日志中不得打印敏感令牌和完整响应体

## 15. 验收清单

- 已确认授权边界和 robots 规则
- 已完成列表页和详情页样本解析
- 已通过字段契约测试
- 已通过去重和聚类回归测试
- 已完成异常页面和空内容处理
- 已接入健康检查和监控
- 已补充样本数据与回归用例
- 已完成后台配置项录入
