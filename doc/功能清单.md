# Trading Studio 功能清单

## 1. 说明

- 优先级说明：`P0` 必做、`P1` 增强、`P2` 后续阶段
- 适用范围：MVP 到商业化一期
- 角色范围：游客、注册用户、会员、专业版、运营管理员、超级管理员

## 4. 系统与数据能力清单

| 领域 | 能力 | 说明 | 优先级 |
| --- | --- | --- | --- |
| 数据源 | Provider 抽象 | 抽象行情、公告、新闻、AI 供应商接口 | P0 |
| 数据源 | 多源切换 | 支持主备 Provider、熔断和降级 | P1 |
| 数据处理 | 去重 | 支持源 ID、URL、标题哈希、SimHash 多层去重 | P0 |
| 数据处理 | 实体识别 | 识别股票、板块、公司、政策部门和产品 | P0 |
| 数据处理 | 事件抽取 | 从新闻和公告生成结构化事件 | P0 |
| 数据处理 | 聚类与合并 | 合并同一事实的多来源新闻和事件 | P1 |
| AI | 结构化输出 | 使用 schema 校验公告解读和事件抽取 | P0 |
| AI | RAG 检索 | 检索公告、新闻、事件、笔记等知识 | P1 |
| AI | 成本统计 | 记录 Token、模型、耗时和失败率 | P0 |
| 通知 | 多通道发送 | 站内信、邮件、微信、Web Push | P1 |
| 搜索 | 全局搜索 | 支持股票、新闻、公告、事件、笔记 | P1 |
| 运维 | 监控告警 | Provider、队列、数据库、模型告警 | P0 |
| 安全 | 内部签名 | Laravel 与 FastAPI 采用 HMAC 请求签名 | P0 |
| 合规 | 输出审查 | AI 输出必须过安全与合规检查 | P0 |

## 5. 版本建议

### 5.1 MVP 必须上线

- 用户系统
- 股票搜索与详情
- 新闻中心
- 公告中心
- 事件中心基础能力
- 自选股
- AI 问答与公告解读
- 后台任务与 Prompt 管理

### 5.2 商业化一期增强

- 多通道通知
- 事件影响链
- 报告导出
- API Token 与开放接口
- 工作区协作
- 成本看板与更精细的运营体系
